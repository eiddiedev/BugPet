# BugPet Website Brief

## Goal

Build a highly restrained, minimal official website for BugPet that primarily serves as a download page.

The website is not meant to be flashy. It should feel calm, clean, confident, and intentional.

## Visual Direction

Keywords:

- restrained
- minimal
- quiet
- native-feeling
- spacious
- confident

Avoid:

- generic startup landing page patterns
- lots of badges or pills
- cluttered hero sections
- multiple equal-priority CTAs
- overloaded gradients
- exaggerated illustrations

## Page Structure

### 1. Hero

Must include:

- BugPet logo
- product name
- one large primary download button
- one short sentence explaining the product

Suggested direction:

- centered layout
- generous whitespace
- little to no decorative chrome

### 2. Features

Briefly explain:

- desktop pet companionship
- coding / focus growth
- contribution heatmap
- TODO panel
- pet system and future expansion

Keep this section concise.

### 3. Usage Guide

Explain basic usage:

- drag the pet
- right-click to open the control panel
- whitelist coding apps
- gain XP while focusing

### 4. Download Notes

Must clearly explain:

- this is a macOS app
- direct download distribution
- no Apple Developer certification / notarization yet
- first launch may require manual trust in macOS settings

Tone requirement:

- honest
- calm
- not defensive

### 5. Repository

Should include:

- GitHub repo link
- open source note
- invitation to contribute

### 6. Developer Panel

Should include:

- developer name: `Eiddie`
- feedback handled via copy-email action rather than exposing the address as large plain text
- GitHub link

## Suggested Copy Tone

Use short, calm sentences.

Good tone:

- “A coding desktop pet for macOS.”
- “Right-click to manage focus, pets, and stats.”
- “Open source and still growing.”

Avoid:

- hype-heavy copy
- exaggerated promises
- long paragraphs above the fold

## Recommended CTA

Primary CTA:

- `Download for macOS`

Optional secondary text below:

- `Open source on GitHub`

## Delivery Recommendation

Best repo structure:

- current repo keeps the app
- new repo hosts the website

Suggested website repo name:

- `BugPet-Web`

## Notes For Future Implementation

If this site is implemented later, prefer:

- static site or lightweight framework
- very small dependency footprint
- easy deployment
- easy screenshot replacement
- easy release version updates
